# 上海华美 AI 咨询智能体（v1.17）

面向 C 端求美者的 AI 售前咨询智能体，目标是替代人工售前咨询：接待求美者 → 了解需求 → 依据院内价目/政策推荐项目/医生 → 引导预约到院面诊。已部署于 Zeabur（GitHub push 自动发布）。

## 运行

```bash
node app/server.js          # 默认 http://localhost:3080，端口被占自动+1
```

### 三种模式（按优先级自动切换：DeepSeek > Claude > 本地）

| 模式 | 条件 | 说明 |
|---|---|---|
| **DeepSeek API 模式** | 设置 `DEEPSEEK_API_KEY` | OpenAI兼容接口；模型用 `DEEPSEEK_MODEL` 指定，网关可用 `DEEPSEEK_BASE_URL` 覆盖 |
| **Claude API 模式** | 设置 `ANTHROPIC_API_KEY` | 模型可用 `CLAUDE_MODEL` 覆盖，默认 claude-sonnet-5 |
| **本地演示模式** | 无任何 API Key | 30+医美意图识别 → 知识库分节检索 → 短句分段应答，可离线演示 |

环境变量：`ACCESS_CODE`（公网部署口令）、`RATE_LIMIT_PER_MIN`（IP限流，默认20次/分钟）、`GEN_TEMPERATURE`（生成温度，默认1.25）。

## 核心机制（四层拟人化架构）

### 0. 生成参数调优
temperature 1.25（DeepSeek 官方闲聊推荐区间 1.2~1.3）、frequency_penalty 0.5、presence_penalty 0.3，解决僵硬感和重复口头禅。Claude 模式下温度上限 1.0，不支持频率/存在惩罚。环境变量 `GEN_TEMPERATURE` 等可调。

### 1. RAG 检索注入 + 对话记忆
- **多轮检索**：按最近 3 条用户消息匹配知识库文档 frontmatter 关键词，只注入最相关的 4 个文档（含目录兜底）。修复了"自评分"导致的检索偏移 bug。
- **客户画像**：从全量历史无状态提取关注项目/预算/顾虑/特殊情况/是否留资 + 滑出窗口的早期消息提要，注入提示词。30 轮后也不忘客户开头说过的话。

### 2. 前端交互仿真（肉眼最直观的"真人感"）
- **流式打字动画**：文本逐字渲染，中文 ~70ms/字、标点 ~30ms，模仿真人打字节奏
- **随机思考延迟**：4 档复杂度（简单问候 2~4s → 复杂需求 7~14s）
- **分段输出切割**：服务端 `|||` 拆分 + 前端逐条播放，条间带自然停顿
- **用户打断**：新消息到达时中止当前回复（AbortController），不再"自顾自说完"
- 发图前"找图"停顿 + 5 种随机话术变体
- 开场接入动画（淘宝客服式"正在接入→弹出顾问"）
- 共情优先：客户流露负面情绪时第一条先接住情绪再谈方案
- system.md 内含对话样例与表情/追问约束

### 3. 模型自主发图
60 张配图目录注入提示词，模型用 `[[图:标签]]` 插图。服务端正则兼容单/双方括号、中英文冒号、单/三竖线分隔。本地模式走关键词匹配。前端兜底剔除残留标记。

### 4. 报价合规
- 价格仅出自知识库；不诊断不承诺疗效
- 孕哺/未成年劝缓；"以到院面诊为准"
- 输出净化：剥离 Markdown 转义、修复穿帮注释外发

### 5. 图片与部署
- assets Q85 压缩（原素材 174MB → 29MB）
- Zeabur CDN 30 天缓存 + immutable + ETag
- 本地开发自动回退素材原图
- IP 限流 20 次/分钟（可调）
