# 腾讯云轻量服务器部署指南

整个流程约 15 分钟：买服务器（5分钟）→ 传文件（2分钟）→ 跑脚本（3分钟）→ 配 Key（1分钟）。

## 一、购买轻量应用服务器（已有可跳过）

1. 打开 https://console.cloud.tencent.com/lighthouse → 新建实例
2. 推荐配置：
   - **地域**：上海（离医院近，访问快）
   - **镜像**：Ubuntu 22.04 LTS（选"系统镜像"，不要选应用镜像）
   - **套餐**：2核2G / 4M 带宽即可（约 ￥50-70/月，新用户常有首年优惠）
3. 创建后在实例详情页记下 **公网IP**，并重置 root 密码（实例 → 更多 → 重置密码）

## 二、放行端口（重要，否则打不开网页）

轻量控制台 → 你的实例 → **防火墙** → 添加规则：

| 应用类型 | 协议 | 端口 | 策略 |
|---|---|---|---|
| 自定义 | TCP | 3080 | 允许 |

## 三、本机打包并上传

在项目根目录（`AI医美咨询Demo`）打开 PowerShell：

```powershell
# 1. 打包（只带代码+知识库+20张配图素材，几十MB）
powershell -ExecutionPolicy Bypass -File deploy\pack.ps1

# 2. 上传 zip 和部署脚本（Windows 10+ 自带 scp；输入 root 密码）
scp deploy\huamei-ai.zip deploy\setup.sh root@服务器公网IP:/opt/
```

> 如果 scp 不可用，也可以用轻量控制台的"文件上传"功能，或安装 WinSCP 拖拽上传到 /opt/。

## 四、登录服务器执行部署脚本

```powershell
ssh root@服务器公网IP
```

登录后：

```bash
bash /opt/setup.sh
```

脚本会自动：装 Node 20 → 解压到 /opt/huamei-ai → 生成环境变量文件 → 注册 systemd 服务（开机自启+崩溃自动重启）。

## 五、配置 DeepSeek Key 和访问口令

```bash
nano /etc/huamei-ai.env
```

改这三行：

```ini
DEEPSEEK_API_KEY=sk-你的key
DEEPSEEK_MODEL=deepseek-chat        # 换成你开通的模型ID
ACCESS_CODE=huamei2026              # 演示口令，发给要看Demo的人；改成你自己的
```

保存（Ctrl+O 回车，Ctrl+X 退出）后重启服务：

```bash
systemctl restart huamei-ai
journalctl -u huamei-ai -n 5    # 看到"模式: DeepSeek API"即成功
```

## 六、访问

浏览器打开 `http://服务器公网IP:3080`，第一次发消息会弹窗要**访问口令**（就是 ACCESS_CODE 设的值），输一次就记住了。

## 常用运维命令

| 操作 | 命令 |
|---|---|
| 看实时日志 | `journalctl -u huamei-ai -f` |
| 重启服务 | `systemctl restart huamei-ai` |
| 改配置 | `nano /etc/huamei-ai.env` 后重启 |
| 更新代码 | 本机重新 `pack.ps1` → `scp` 上传 → 服务器 `bash /opt/setup.sh`（配置不会被覆盖） |

## 可选进阶

- **用 80 端口**（省得带端口号）：`/etc/huamei-ai.env` 里 `PORT=80`，防火墙放行 80，重启服务。
- **绑域名 + HTTPS**：域名解析到服务器 IP 后，装 Caddy 一行反代自动 HTTPS。⚠️ 域名在大陆服务器上对外提供 Web 服务需要 **ICP 备案**（腾讯云控制台可免费办理，约 1-2 周）；用 IP:端口 访问则不需要。
- **正式上线提醒**：面向公众运营前，AI 生成内容标识、医疗广告合规请让院方法务确认。
